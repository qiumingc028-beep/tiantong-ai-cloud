# V2 Skill Versioning Policy

日期：2026-07-12

## 1. 原则

Skill 发布后版本不可原地覆盖。任何修改必须生成新版本。

## 2. 版本生命周期

创建 Skill → 提交 Manifest → 审核 → 发布版本 → 安装 → 启用 → 调用 → 升级 → 停用 → 撤销 → 归档

## 3. 版本规则

- 历史版本永久保留
- 可查看版本差异
- 可回退，但回退也必须形成新安装记录
- 被调用过的版本不可物理删除
- 升级必须生成新版本

## 4. 安装与回退

安装记录必须保留：

- 安装员工
- 安装版本
- 安装配置
- 权限快照
- 签名与校验结果

回退只能通过新安装记录实现，不能直接覆盖旧版本状态。

## 5. 安全要求

- 未签名第三方 Skill 默认禁止启用
- 自动更新默认禁止
- 风险提升必须重新审批
- 版本兼容性必须校验

## 6. 结论

Skill 版本体系以“可追溯、不可覆盖、可审计”为唯一目标，确保技能演进不破坏历史可信度。
