# TIANTONG AI V2 能力中心规范

## 1. 能力中心目标

能力中心是 V2 的统一能力目录，用于管理 AI 员工可调用的所有能力。它只负责“定义、展示、开关、审批、审计”，不直接执行真实工具。

## 2. 能力注册字段

每项能力至少包含：

- 唯一标识
- 中文名称
- 中文说明
- 能力类型
- 执行器类型
- 风险等级
- 是否启用
- 是否只读
- 是否需要老板审批
- 是否需要安全审计
- 超时时间
- 最大重试次数
- 输入 Schema
- 输出 Schema
- 允许使用的员工
- 创建时间
- 更新时间
- 版本号

## 3. 能力类型枚举

预留能力类型包括：

- 数据读取
- API 调用
- 浏览器操作
- 电脑操作
- 手机操作
- 文件处理
- 数据库操作
- Shell 操作
- Git 操作
- Docker 操作
- 邮件操作
- 日历操作
- 搜索能力
- AI 模型调用

## 4. 风险等级

建议统一分为：

- 低风险：只读、可自动执行
- 中风险：需要明确权限
- 高风险：需要老板审批
- 极高风险：默认禁止

## 5. 默认能力

本 Sprint 只落地安全模拟能力：

- `mock.echo`
- `mock.failure`
- `mock.timeout`
- `mock.retry`
- `task.writeback`

这些能力不访问真实外部系统，不执行 Shell，不控制电脑、浏览器或手机。

## 6. 能力中心页面

页面名称：`能力中心`

页面只显示：

- 能力名称
- 能力类型
- 风险等级
- 是否启用
- 是否只读
- 是否需要审批
- 执行器状态

主操作：

- `查看能力详情`

页面原则：

- 极简
- 中文
- 老板一眼能看懂
- 不展示数据库字段
- 不展示 JSON
- 少按钮、少表格

## 7. 执行记录页面

页面名称：`执行记录`

展示：

- 执行任务
- AI 员工
- 使用能力
- 当前状态
- 风险等级
- 审批状态
- 开始时间
- 执行耗时
- 执行结果

详情展示：

- 执行流程
- 审批记录
- 审计记录
- 错误原因
- 重试记录

敏感字段必须脱敏。

## 8. 权限与审批

能力调用前必须检查：

- 员工是否存在
- 员工是否启用
- 员工是否拥有该能力
- 能力是否启用
- 能力是否只读
- 风险等级
- 是否需要老板审批
- 是否需要安全审计
- 当前环境是否允许该执行器
- 是否超过频率限制
- 输入是否符合 Schema

默认策略：

- 低风险只读模拟能力：允许自动执行
- 中风险能力：需要明确权限
- 高风险能力：必须老板审批
- 极高风险能力：默认禁止

## 9. 生产安全策略

生产环境默认：

- 只允许模拟执行器
- 禁止真实电脑控制
- 禁止真实手机控制
- 禁止真实浏览器控制
- 禁止 Shell 执行

## 10. 未来扩展

后续 OpenClaw、Browser、Desktop、Mobile、Shell、Git、Docker 以及更多工具，都应作为能力接入，不应绕过能力中心与审批链。
