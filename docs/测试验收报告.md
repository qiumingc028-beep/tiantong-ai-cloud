## 风险项

1. 上传文件大小限制不足
   - 现状：`/api/metrics/import` 会读取上传文件内容，但没有发现明确的大小上限。
   - 影响：恶意或误操作上传大文件时，可能造成内存压力。
   - 建议：在业务模块负责人确认后，为导入接口增加文件大小限制，例如 5MB 或 10MB，并返回 HTTP `413`。

2. `/api/jd/metrics/summary` 权限较宽
   - 现状：测试记录了低权限登录用户也能读取汇总数据，这是当前实现行为。
   - 影响：如果汇总经营数据只允许老板或京东数据权限用户查看，当前权限边界偏宽。
   - 建议：由业务负责人确认该接口是否应要求 `menu.dashboard` 或 `menu.jd_data` 权限。

3. 登录接口返回 token
   - 现状：登录响应体会返回 token，同时也设置 HttpOnly Cookie。
   - 影响：这是常见 API 设计，但前端如果把 token 写入不安全存储，会增加泄露风险。
   - 建议：上线前确认前端不把 token 写入 localStorage；优先使用 HttpOnly Cookie。

4. 默认示例密码需要生产替换
   - 现状：`.env.example` 和 Docker 示例中有默认数据库密码、默认 JWT 示例值。
   - 影响：如果生产环境照抄默认值，会造成高风险。
   - 建议：上线前确认生产 `.env` 使用强密码和强 `JWT_SECRET`，且不提交仓库。
