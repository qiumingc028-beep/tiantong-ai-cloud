# 天检测试清单

状态：准备阶段，等待天采、天册、天颜、天盾完成后执行最终验收。

原则：不写假通过。每一项最终验收时必须记录实际命令、实际结果、失败原因和处理建议。

## 一、基础接口

### 1. 健康检查

- 测试对象：`GET /api/health`
- 测试命令：

```bash
curl -i http://127.0.0.1:8000/api/health
```

- 预期结果：HTTP `200`，返回 `status=running`，数据库和 Redis 状态正常。
- 重点风险：数据库不可达、Redis 不可达、健康接口隐藏真实故障。
- 当前状态：待最终验收。

### 2. 就绪检查

- 测试对象：`GET /api/ready`
- 测试命令：

```bash
curl -i http://127.0.0.1:8000/api/ready
```

- 预期结果：HTTP `200`，返回 `status=ready`。
- 重点风险：服务启动但数据库或 Redis 未就绪。
- 当前状态：待最终验收。

## 二、登录与权限

### 3. 登录成功

- 测试对象：`POST /api/login`
- 测试命令：

```bash
curl -i -X POST http://127.0.0.1:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"owner","password":"password"}'
```

- 预期结果：HTTP `200`，返回 `ok=true`，设置 HttpOnly Cookie。
- 重点风险：响应泄露 `password`、`password_hash`、敏感 Cookie 属性不完整。
- 当前状态：待最终验收。

### 4. 未登录访问拦截

- 测试对象：所有需登录接口。
- 测试命令：

```bash
curl -i http://127.0.0.1:8000/api/me
curl -i http://127.0.0.1:8000/api/accounts
curl -i http://127.0.0.1:8000/api/templates
curl -i http://127.0.0.1:8000/api/brands
curl -i http://127.0.0.1:8000/api/store-groups
```

- 预期结果：未登录时返回 HTTP `401`。
- 重点风险：接口可匿名读取经营、账号或模板资料。
- 当前状态：待最终验收。

### 5. 低权限越权检查

- 测试对象：新增账号资料中心接口。
- 测试命令：

```bash
curl -i -H "Authorization: Bearer <低权限账号token>" http://127.0.0.1:8000/api/accounts
curl -i -H "Authorization: Bearer <低权限账号token>" http://127.0.0.1:8000/api/templates
curl -i -H "Authorization: Bearer <低权限账号token>" http://127.0.0.1:8000/api/brands
curl -i -H "Authorization: Bearer <低权限账号token>" http://127.0.0.1:8000/api/store-groups
```

- 预期结果：无对应权限时返回 HTTP `403`。
- 重点风险：普通员工越权读取或修改账号资料。
- 当前状态：待最终验收。

## 五、Excel 导入导出

### 14. 账号资料 Excel 导入

- 测试命令：

```bash

  -H "Authorization: Bearer <owner_token>" \
  -F "file=@docs/templates/jd_60_store_import_template.csv"
```

- 预期结果：如果接口只接受 `.xlsx`，CSV 应返回 HTTP `400` 且提示格式错误；使用合法 `.xlsx` 时返回 `ok=true` 和导入数量。

## 六、安全专项

### 17. 明文密码检查

- 测试对象：数据库账号资料表、用户表、导入导出文件。
- 测试命令：

```bash
python -m pytest tests/test_auth.py -q
```

- 预期结果：用户密码只保存哈希；账号资料接口和导出文件不出现明文密码。
- 重点风险：`password`、`encrypted_password` 字段实际保存明文。
- 当前状态：待最终验收。

### 18. Cookie 泄露检查

- 测试对象：页面源码、接口响应体、导出文件。
- 测试命令：

```bash
curl -s -H "Authorization: Bearer <owner_token>" http://127.0.0.1:8000/api/accounts | grep -Ei "cookie|token|password|secret|key"
```

- 预期结果：不得返回原始 Cookie、token、密码、API Key。
- 重点风险：京东账号 Cookie 或采集凭证被接口返回给前端。
- 当前状态：待最终验收。

### 19. 页面跳转链路检查

- 测试对象：首页、菜单、账号资料中心、模板中心、品牌中心、店铺分组。
- 测试命令：

```bash

curl -i http://127.0.0.1:8000/template-center.html
curl -i http://127.0.0.1:8000/brands.html
curl -i http://127.0.0.1:8000/store-groups.html
```

- 预期结果：页面都返回 HTTP `200`，页面内导航不出现死链。
- 重点风险：天颜完成后菜单入口和页面路由不一致。
- 当前状态：待最终验收。
