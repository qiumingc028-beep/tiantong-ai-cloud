# V2 执行可观测性与安全运营架构

## 目标

建立统一的设备运行监控、执行质量评分、风险评分、异常检测与安全运营中心。
系统只做观测、评分、告警与有限自动处置，不扩大电脑控制能力，不接入真实生产业务。

## 分层

1. 设备与会话层：Device、Computer Session、Workflow、Action、Replay。
2. 指标采集层：采集设备运行指标、执行运行指标、设备健康评分、员工执行绩效。
3. 评分引擎层：输出可解释的质量评分和风险评分。
4. 异常检测层：基于确定性规则生成异常事件。
5. 安全事件层：记录安全事件、告警规则、熔断器和处理状态。
6. 运营展示层：安全运营中心、设备监控、执行质量、安全事件。

## 安全边界

- 只读取已授权测试设备和已存在的执行数据。
- 不读取剪贴板、Cookie、用户个人文件、密码、验证码。
- 不执行 Shell，不控制真实电脑，不扩大工作流步骤上限。
- 回放索引只读，不能反向驱动执行。
- 普通用户不能查看或重置安全事件、熔断器和告警规则。

## 自动处置

默认只允许：

- 暂停会话
- 暂停工作流
- 取消待执行审批
- 禁用测试设备
- 撤销设备凭据
- 请求人工接管
- 生成告警

不允许：

- 删除安全事件
- 自动恢复高风险熔断
- 修改生产系统
- 自动执行 Shell

## Feature Flag

所有新增能力默认关闭，生产环境默认关闭。
