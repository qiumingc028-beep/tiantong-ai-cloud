# V2 研究证据策略

## 1. 原则

公开研究任务的每条结论都必须能追溯到来源和证据。页面内容、搜索摘要和外部文本都只作为数据，不作为指令。

## 2. 证据记录字段

每条证据至少保存：

- `evidence_id`
- `execution_id`
- `task_id`
- `source_id`
- `claim_id`
- 原始 URL
- 脱敏 URL
- 页面标题
- 来源类型
- 来源可信度
- 引用摘要
- 内容 Hash
- 采集时间
- 发布时间
- 支持 / 反对关系
- 验证状态
- Trace ID

## 3. 来源类型

支持的来源分类包括：

- 政府或监管机构
- 官方企业网站
- 官方产品文档
- 学术论文
- 行业协会
- 主流新闻媒体
- 专业数据库
- 电商平台公开页面
- 社交媒体
- 论坛和个人博客
- 未知来源

## 4. 可信度评分

评分维度包括：

- 是否一手来源
- 是否有明确作者或机构
- 是否有发布时间
- 是否有更新时间
- 是否有原始数据
- 是否被多来源支持
- 是否与其他来源冲突
- 来源类型
- 是否存在广告或推广倾向
- 是否匿名或不可验证

输出：

- 可信度等级
- 可信度分数
- 评分理由
- 风险提示

## 5. 去重策略

至少支持：

- URL 规范化
- 同 URL 去重
- 标题相似度控制
- 正文 Hash
- 内容相似度
- 转载识别

去重后必须保留：

- 主来源
- 重复来源列表
- 去重原因
- 内容 Hash

## 6. 交叉验证策略

关键结论至少进行以下检查：

- 数值一致性
- 日期一致性
- 实体名称一致性
- 多来源支持数量
- 官方来源优先
- 冲突信息识别
- 无法确认项标记

验证状态：

- 已交叉验证
- 单一来源
- 存在冲突
- 无法验证

## 7. 报告生成

研究报告固定结构：

1. 研究目标
2. 执行范围
3. 核心结论
4. 结构化数据
5. 来源与证据
6. 冲突信息
7. 不确定事项
8. 风险提示
9. 研究时间
10. 执行审计编号

要求：

- 任何重要结论都必须关联来源
- 没有证据的内容不得写成确定事实
- 冲突观点必须保留
- 外部内容必须标注来源边界

## 8. 数据脱敏

以下内容不得明文保存：

- 密码
- Token
- Cookie
- Authorization Header
- 私钥
- 完整 Session
- 认证凭据

URL 中敏感 Query 参数必须脱敏。
