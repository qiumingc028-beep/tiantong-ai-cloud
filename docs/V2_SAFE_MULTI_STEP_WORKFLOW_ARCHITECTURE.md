# V2 安全多步骤工作流架构

## 目标

本工作流只用于测试环境中的有限多步骤桌面操作编排，解决“单步安全动作可以执行，但无法组合成受控流程”的问题。

## 分层

1. 老板 / Orchestrator 创建测试任务。
2. AI 员工通过 Skill 申请工作流。
3. Skills Engine 校验 Skill、版本、授权与风险。
4. Agent Runtime 创建并管理工作流。
5. 设备、会话、应用和窗口白名单校验。
6. 工作流计划生成与范围审批。
7. 每步动作执行前校验。
8. 每步动作执行后验证。
9. 关键节点审批。
10. Task Center 回写与审计记录。

## 核心约束

- 最大步骤数固定为 5。
- 最小步骤数固定为 2。
- 工作流计划在执行前一次性生成。
- 计划变化后，原审批自动失效。
- 每一步执行后都必须验证并自动暂停。
- 关键节点必须人工确认。
- 失败即停止，不自动扩展步骤。

## 安全边界

- 不允许 Terminal、Shell、剪贴板、文件上传、文件下载。
- 不允许真实业务系统操作。
- 不允许连续无限自动运行。
- 不允许未经审批执行未知动作。
- 生产环境默认关闭。
