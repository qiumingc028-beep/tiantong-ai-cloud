## 2. 验收角色

模拟角色：

```text
老板第一次打开 AI员工中心
```

验收标准：

- 5 秒内看懂
- 不需要培训
- 不需要理解技术词
- 能完成：首页 → 员工卡片 → 员工详情 → 返回

## 3. 场景 1：首页首次打开

### 3.1 是否知道这里是什么

结论：通过。

页面第一眼能看到：

- `AI员工中心`
- `我的AI公司现在怎么样？`

老板可以判断这是查看 AI员工状态的页面。

### 3.2 是否知道有多少 AI员工

结论：通过。

首页第一屏展示：

- `AI员工`
- 数量位置：`employeeTotal`

这符合“先看总数”的使用目标。

### 3.3 是否知道哪些员工正在工作

结论：通过。

首页第一屏展示：

- `正在工作`
- 数量位置：`workingTotal`

员工卡片也展示员工当前状态，例如：

- 正在工作
- 正常待命
- 需要老板看看
- 暂时离线

### 3.4 是否知道怎么查看员工

结论：通过。

页面有明确提示：

- `下面是你的AI员工`
- `点卡片看详情`
- `看详情`

老板能自然点击员工卡片进入详情。

## 4. 场景 2：点击一个 AI员工

### 4.1 是否理解这个 AI员工负责什么

结论：通过。

详情页有明确模块：

- `我是谁`
- `我帮公司做什么`

页面使用普通语言，不需要理解技术背景。

### 4.2 是否理解今天做了什么

结论：通过。

详情页有明确模块：

- `今天完成什么`

页面展示：

- 今天完成几件事
- 几件事等待老板确认
- 几件事需要复盘

### 4.3 是否理解有没有成长

结论：通过。

详情页有明确模块：

- `我的成长`

页面展示：

- 成长分
- 新经验
- 复盘经验

这是老板能理解的表达。

### 4.4 是否能返回员工中心

结论：通过。

详情页有：

```text
返回AI员工中心
```

用户闭环成立。

## 5. 场景 3：技术信息与复杂度检查

### 5.3 按钮数量

结论：通过。

主要动作只有：

- 看详情
- 返回AI员工中心

### 5.4 文字数量

结论：基本通过。

首页文字较少，符合要求。

详情页下方仍保留：

这些信息不影响首屏理解，但后续可以继续折叠或放到“更多”区域。

## 6. 用户体验问题报告

发现 2 个轻微体验问题，不影响 V1 验收：

### 问题 1：详情页下半部分仍略像资料页

现象：

- `我还会什么`
- `成长记录`
- `经验记录`

这些内容对老板有价值，但对第一次使用者不是第一优先级。

建议：

- 后续版本可默认折叠。
- 首屏只保留四个核心模块。

### 问题 2：顶部侧边导航仍有系统感

现象：

- 左侧仍显示 `企业大脑`、`AI员工中心`

这不复杂，但仍偏系统页面。

建议：

- V2 可以考虑把 AI员工中心做成更像通讯录的独立页面。
