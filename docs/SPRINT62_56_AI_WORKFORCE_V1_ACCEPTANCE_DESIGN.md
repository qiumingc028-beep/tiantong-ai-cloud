# Sprint62.56 AI员工中心 V1 整合验收设计

## 1. 目标

把 AI员工首页、员工详情、成长记录、能力展示统一成一个老板能看懂的简单入口。

核心问题：

> 我的 AI 公司现在怎么样？

设计原则：

- 少
- 清楚
- 简单
- 三岁小孩可以理解
- 像苹果产品一样先给结论，再给细节

本阶段只做设计，不写代码，不修改数据库，不接入自动执行。

## 4. 信息层级

### 4.1 第一层：一句话结论

老板最先看到：

```text
AI员工正常运行
```

或：

```text
有事项等待Boss确认
```

或：

```text
有AI员工需要关注
```

这一层只回答：

> 现在好不好？

### 4.2 第二层：三个数字

展示：

- AI员工数量
- 今日任务
- 平均成长评分

这一层回答：

> 公司规模多大？今天做了多少？成长怎么样？

### 4.3 第三层：员工卡片

展示：

- 员工名称
- 部门
- 当前状态
- 成长评分

这一层回答：

> 哪些 AI员工在工作？

### 4.4 第四层：员工详情

展示：

- 我是谁
- 我最近做了什么
- 我学会了什么
- 我的成长

这一层回答：

> 这个 AI员工到底有什么用？

### 4.5 第五层：成长和能力

展示：

- 技能卡片
- 成长时间线
- 成长评分
- 经验记录

这一层回答：

> 它怎么变强了？

## 5. 哪些内容删除

V1 应删除或隐藏：

- 数据库字段
- API字段
- employee_id
- 原始 JSON
- 技术日志
- 技术状态
- 密集表格
- 后台管理式筛选
- 复杂菜单
- 任务执行按钮
- 技能升级按钮
- 权限修改入口
- 调用 Skill 的入口
- Execution Engine 入口
- OpenClaw 入口
- n8n 入口

原因：

老板入口不是后台管理台。老板只需要知道状态、能力、成长和风险。

```text
AI员工中心
```

V1 页面命名建议：

```text
AI员工中心

```

```text
AI Workforce Center
Growth System
Skill API
Memory API
```

## 9. 验收标准

### 9.1 视觉验收

通过条件：

- 第一屏 3 秒内知道 AI公司是否正常。
- 员工展示不是表格，是卡片。
- 员工详情不是后台字段，是自然语言。
- 页面少文字、少模块、少按钮。

### 9.2 内容验收

通过条件：

- 首页显示 AI员工数量、运行状态、今日任务、平均成长。
- 员工卡片显示名称、部门、状态、成长评分。
- 详情页显示我是谁、最近做什么、学会什么、我的成长。
- 成长记录显示时间线、评分、经验。
- 能力展示显示技能卡片、等级、使用次数、成功率。

### 9.3 安全验收

通过条件：

- 无执行按钮
- 无 POST 调用
- 无 Task Center 写入口
- 无 Execution Engine 接入
- 无 OpenClaw 接入
- 无 n8n 接入
- 无技能自动升级
- 无权限修改入口
- 保持 `readonly=true`
- 保持 `boss_confirm=true`
- 保持 `security_audited=true`

