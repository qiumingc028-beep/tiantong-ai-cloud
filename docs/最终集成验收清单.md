## 二、最终验收测试清单

### 1. 登录

- 测试对象：`POST /api/login`
- 测试命令：

```bash
curl -i -X POST http://127.0.0.1:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"owner","password":"password"}'
```

- 必须通过：HTTP `200`，返回 `ok=true`，设置 HttpOnly Cookie，响应体不包含 `password_hash`。
- 失败排查：账号密码、用户状态、数据库连接、Redis 会话写入。

### 2. 退出登录

- 测试对象：`POST /api/logout`
- 测试命令：

```bash
curl -i -X POST http://127.0.0.1:8000/api/logout \
  -b "tiantong_session=<登录Cookie>"
```

- 必须通过：HTTP `200`，返回 `ok=true`，Cookie 被清理；清理后访问 `/api/me` 返回 `401`。
- 失败排查：Cookie 删除逻辑、Redis session 删除、前端退出跳转。

### 3. 老板驾驶舱

## 三、重点安全检查清单

### 1. `.env` 是否泄露

- 检查命令：

```bash
test ! -f .env
git status --short
```

- 必须通过：仓库不提交 `.env`；生产环境密钥只在服务器环境变量或安全配置中。

### 2. Token 是否泄露

- 检查命令：

```bash
curl -s -H "Authorization: Bearer <owner_token>" http://127.0.0.1:8000/api/me
curl -s -H "Authorization: Bearer <owner_token>" http://127.0.0.1:8000/api/accounts
```

- 必须通过：除登录接口明确返回 token 外，其他业务接口不得返回 token、refresh token、access token。

### 3. Cookie 是否泄露

- 检查命令：

```bash
curl -s -H "Authorization: Bearer <owner_token>" http://127.0.0.1:8000/api/accounts
curl -i -H "Authorization: Bearer <owner_token>" http://127.0.0.1:8000/api/accounts/export
```

- 必须通过：接口和导出文件不得返回京东 Cookie、浏览器 Cookie、会话 Cookie。

### 4. 是否有明文密码

- 检查命令：

```bash
python -m pytest tests/test_auth.py -q
curl -s -H "Authorization: Bearer <owner_token>" http://127.0.0.1:8000/api/accounts
```

- 必须通过：用户密码为哈希；账号资料接口不返回明文密码；导入导出不产生明文密码。

### 5. Excel 上传是否有限制

- 检查命令：

```bash
python -m pytest tests/test_import.py -q
```

- 必须通过：非法类型、空文件、超大文件被拒绝；合法文件正常导入。

### 6. 普通员工是否能越权访问老板接口

- 检查命令：

```bash
curl -i -H "Authorization: Bearer <普通员工token>" http://127.0.0.1:8000/api/dashboard
curl -i -H "Authorization: Bearer <普通员工token>" http://127.0.0.1:8000/api/owner/dashboard
curl -i -H "Authorization: Bearer <普通员工token>" http://127.0.0.1:8000/api/accounts
```

- 必须通过：无老板或管理权限时返回 `403`，不得返回经营汇总或账号资料。

## 四、风险清单

1. Excel 上传大小限制尚未形成完整通过项
   - 风险：大文件可能造成内存压力。
   - 必须处理：最终验收前确认 `/api/metrics/import` 和 `/api/accounts/import` 均有大小限制。

2. 新增账号资料中心接口尚未全部自动化覆盖
   - 风险：`/api/accounts`、`/api/templates`、`/api/brands`、`/api/store-groups` 可能存在权限或敏感字段遗漏。
   - 必须处理：最终验收前补自动化或执行手工验收记录。

3. Excel 导出敏感字段风险
   - 风险：导出文件可能包含明文密码、Cookie、Token、API Key。
   - 必须处理：最终验收必须下载文件并检查内容。

4. 普通员工越权访问老板接口风险
   - 风险：如果老板驾驶舱接口只要求登录而不要求老板权限，普通员工可能读取经营汇总。
   - 必须处理：最终验收必须用普通员工账号验证。

5. worker 和 Redis 队列未纳入现有 pytest
   - 风险：接口正常但后台任务不消费。
   - 必须处理：最终验收必须验证 worker 状态、Redis 队列和任务状态记录。

6. PostgreSQL 迁移需在真实数据库验证
   - 风险：SQLite 测试通过不代表 PostgreSQL 迁移通过。
   - 必须处理：最终验收必须执行 Alembic 迁移检查。

7. Nginx 页面访问需在部署入口验证
   - 风险：后端 8000 正常但 80 端口页面或 API 转发异常。
   - 必须处理：最终验收必须通过 Nginx 入口访问页面和 API。
