# V2 人工接管策略

## 目标

当电脑执行器遇到敏感区域、异常窗口、白名单外应用或高风险步骤时，必须支持暂停并请求人工接管。

## 默认状态

- HUMAN_TAKEOVER_ENABLED = false

## 触发条件

- 敏感输入框
- 密码 / Token / 验证码
- 无法识别窗口
- 风险升级
- 会话状态异常

## 接管要求

1. AI 必须暂停后续动作。
2. 记录接管请求和审批状态。
3. 接管期间不得继续自动输入。
4. 接管结束后可选择继续或终止。

## 安全边界

- 不记录明文敏感输入。
- 不允许接管机制绕过审批。
- 不允许接管过程升级为真实 Shell 或真实桌面控制。
