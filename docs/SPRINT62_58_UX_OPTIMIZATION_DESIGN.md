# Sprint62.58 AI员工中心 V1 用户体验优化设计

## 1. 目标

把 AI员工中心从“开发者能看懂”优化为“老板第一次打开就会用”。

页面第一眼只回答一个问题：

```text
我的AI公司现在怎么样？
```

## 2. 已读取文件

- `README.md`
- `docs/SPRINT62_57_ACCEPTANCE_REPORT.md`
- `frontend/ai-workforce.html`

## 3. 优化原则

1. 少信息：第一屏只保留公司状态、员工数量、已完成任务、正在工作的员工。
2. 少术语：页面不展示 API、数据库、ID、状态码、技术字段。
3. 少操作：只提供查看入口，不提供执行、升级、授权、自动运行入口。
4. 颜色直观：
   - 绿色 = 正常
   - 黄色 = 注意
   - 红色 = 问题
5. 文字直白：
   - “正在工作”
   - “已经完成”
   - “需要老板确认”

## 4. 首页信息层级

### 第一屏：AI公司状态

保留：

- AI员工运行状态
- AI员工数量
- 已经完成的任务数量
- 正在工作的员工数量

删除或隐藏：

- 技术状态
- API字段
- 数据库字段
- 状态码
- 复杂成长评价术语

### 第二屏：AI员工列表

员工卡片只展示：

- 头像
- 名字
- 负责什么
- 当前状态
- 进入员工详情

不展示：

- employee_id
- 技术字段
- 技能明细堆叠
- 当前任务技术字段
- 风险处理按钮

### 第三层：员工详情入口

保留：

```text
进入员工详情
```

跳转到现有员工详情页。

不新增执行按钮，不新增任务创建按钮，不新增权限修改入口。

## 5. 小范围前端调整

本阶段对 `frontend/ai-workforce.html` 做小范围 UX 调整：

- Sprint 标识更新为 Sprint62.58。
- 顶部安全字段从技术文本改为老板可理解文本：
  - `readonly=true` 展示为“只读模式”
  - `boss_confirm=true` 展示为“老板确认”
  - `security_audited=true` 展示为“安全审计”
- 安全标记保留在页面属性中，用于测试和审计。
- 第一屏指标改为：
  - AI员工
  - 运行状态
  - 已经完成
  - 正在工作
- 员工卡片只保留：
  - 头像
  - 名称
  - 部门
  - 当前状态
  - 负责事项
  - 进入员工详情

## 6. 安全边界

保持：

- `readonly=true`
- `boss_confirm=true`
- `security_audited=true`

禁止：

- 自动执行
- POST 接口
- 修改 Task Center
- 修改数据库
- 创建 migration
- 接入 Execution Engine
- 接入 OpenClaw
- 接入 n8n
- 自动升级员工
- 自动修改权限

## 7. 验收标准

1. 首页第一眼可以看懂 AI公司状态。
2. 员工卡片只展示老板关心的基础信息。
3. 页面不出现 API、ID、数据库、状态码等技术语言。
4. 不出现执行、升级、授权、自动运行入口。
5. 相关前端测试通过。

## 8. 后续建议

Sprint62.59 可以继续做：

- 员工详情页同样老板化表达。
- AI员工中心入口与企业大脑总控台入口统一。
- Viewer 视角下的信息降级展示。

仍不建议进入自动执行、自动技能调用或权限自动化。
