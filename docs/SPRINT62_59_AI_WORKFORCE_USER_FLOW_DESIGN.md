# Sprint62.59 AI员工中心 V1 老板使用闭环设计

## 1. 阶段边界

本阶段只做产品使用闭环设计，不写代码。

禁止：

- 修改数据库
- 创建 migration
- 修改 Task Center 核心逻辑
- 接入 Execution Engine
- 接入 OpenClaw
- 接入 n8n
- 增加自动执行能力

## 2. 已读取文件

- `README.md`
- `docs/SPRINT62_58_UX_OPTIMIZATION_DESIGN.md`
- `frontend/ai-workforce.html`

## 3. 设计目标

让老板第一次打开 AI员工中心，不需要培训，也能完成一个完整查看闭环：

```text
首页
↓
点击AI员工
↓
进入员工详情
↓
看懂这个员工负责什么、今天做了什么、学到了什么、目前状态
↓
返回员工中心
```

核心问题只有一个：

```text
我的AI公司现在怎么样？
```

## 4. 用户路径设计

### 4.1 第一步：首页看公司状态

老板打开 `frontend/ai-workforce.html`。

第一屏只展示：

- AI员工是否正常
- 一共有多少 AI员工
- 今天已经完成多少任务
- 现在有多少员工正在工作

首页不展示：

- API
- ID
- 数据库字段
- 状态码
- 技术字段
- 复杂后台菜单

### 4.2 第二步：看 AI员工卡片

员工卡片只回答四件事：

```text
它是谁？
负责什么？
现在怎么样？
能不能点进去看详情？
```

卡片字段：

- 头像
- 名字
- 负责什么
- 当前状态
- 进入员工详情

状态文案：

- 绿色：正在工作 / 正常待命
- 黄色：需要老板看看
- 红色：暂时离线 / 有问题

### 4.3 第三步：进入员工详情

点击：

```text
进入员工详情
```

进入现有员工详情页。

详情页第一屏只回答：

```text
这个AI员工现在值不值得放心？
```

推荐信息顺序：

1. 员工名称
2. 所属部门
3. 当前状态
4. 负责什么
5. 今天做了什么
6. 学到了什么
7. 目前是否需要老板关注

### 4.4 第四步：看懂员工详情

员工详情页建议拆成四个简单区域。

#### 区域一：这个员工负责什么

展示：

- 一句话岗位说明
- 所属部门
- 主要职责

示例：

```text
天采负责收集和整理电商经营数据。
```

#### 区域二：今天做了什么

展示：

- 今天完成了多少事
- 最近一件事是什么
- 是否有待老板确认事项

文案使用：

- 已经完成
- 正在处理
- 需要老板确认

不使用：

- task lifecycle
- queue status
- transition state

#### 区域三：学到了什么

展示：

- 新增经验
- 技能变化
- 复盘记录

文案使用：

- 学会了什么
- 哪方面更熟练
- 有什么经验沉淀

不使用：

- growth evaluation
- model update
- vector memory

#### 区域四：目前状态

展示：

- 正常
- 需要注意
- 有问题

颜色规则：

- 绿色 = 正常
- 黄色 = 注意
- 红色 = 问题

## 5. 返回员工中心

详情页顶部或底部保留一个清晰返回入口：

```text
返回AI员工中心
```

返回后老板继续查看其他员工。

不增加：

- 多层面包屑
- 复杂菜单
- 后台管理入口
- 执行按钮

## 6. 页面闭环结构图

```text
AI员工中心首页
  ├─ AI公司状态
  ├─ AI员工卡片
  │   ├─ 天采
  │   ├─ 天数
  │   ├─ 天商
  │   └─ 其他AI员工
  ↓
员工详情页
  ├─ 我是谁
  ├─ 我负责什么
  ├─ 我今天做了什么
  ├─ 我学到了什么
  ├─ 我现在怎么样
  ↓
返回AI员工中心
```

## 7. 信息保留与删除

### 保留

- AI公司整体状态
- 员工数量
- 今天已完成任务
- 正在工作的员工
- 员工名称
- 员工职责
- 员工状态
- 进入员工详情
- 返回员工中心

### 删除或隐藏

- API 字段
- 数据库字段
- employee_id
- 状态码
- 技术日志
- 技术状态
- 密集表格
- 复杂筛选
- 多级后台菜单

## 8. 操作设计

V1 只允许两个动作：

1. 查看员工详情
2. 返回员工中心

禁止出现：

- 执行
- 启动
- 自动运行
- 自动升级
- 修改权限
- 自动分配任务
- 自动调用技能

## 9. 安全边界

保持：

- `readonly=true`
- `boss_confirm=true`
- `security_audited=true`

任何高风险变化未来都必须：

```text
boss_confirm=true
security_audited=true
```

本阶段不接入：

- Execution Engine
- OpenClaw
- n8n

## 10. 后续开发建议

Sprint62.60 可以按最小范围实现：

1. 员工详情页增加“返回AI员工中心”入口。
2. 员工详情页文案老板化。
3. 员工详情页四个区域收敛为：
   - 负责什么
   - 今天做了什么
   - 学到了什么
   - 目前状态
4. 增加静态测试，确认无执行按钮、无技术字段、无高风险入口。

不建议在下一阶段加入自动执行、自动任务创建、自动技能调用或权限修改能力。
