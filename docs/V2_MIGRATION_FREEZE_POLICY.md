# V2 Migration Freeze Policy

## 1. 冻结范围

1. `0001`—`0027` 属于 V1 正式发布链，永久冻结，不得修改。
2. `0028`—`0043` 属于 V2 预发布链，当前允许在 Sprint 11.1 收口期做最后一次证据整理，但不得再引入新业务功能。
3. `0037` 的布尔默认值调整属于 V2 预发布兼容修正，用于保证 PostgreSQL 实库可执行性。
4. `knowledge_asset_id` 不作为 Alpha Workflow 的全局唯一身份，仅保留普通索引。
5. `0043_v2_alpha_workflow_knowledge_asset_identity_repair.py` 是 forward-only 例外修复，用于把 `knowledge_asset_id` 从全局唯一性中移除。

## 2. 收口后冻结

Sprint 11.1 关闭后：

- `0037`—`0043` 与其它已发布迁移一样冻结。
- 后续只允许新增 Migration，不允许修改已冻结文件。
- 不允许通过修改旧文件掩盖历史差异。

## 3. 生产与环境边界

- 当前 V2 Migration 未执行于生产环境。
- 证据与验证只允许在隔离 PostgreSQL 环境中完成。
- 不允许使用 SQLite 作为正式证据，也不允许使用 drift 跳过开关。

## 4. 变更审批要求

迁移文件的任何后续修改，必须满足：

- ① 实施人明确记录修改文件与原因
- ③ 验收人确认回归结果
- ④ 审查人给出最终审查结论

## 5. 允许的后续动作

- 仅可通过新增 Migration 修复后续发现的问题。
- 证据文件可以更新，以反映真实执行结果。
- 旧 Migration 一旦冻结，不再回写。
