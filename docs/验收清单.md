# 天统AI云中台验收清单

本清单用于真实验收，不写假通过。每一项都需要按命令执行，并以实际输出为准。

默认地址为 `http://127.0.0.1:8000`，默认账号为 `owner`，默认密码为 `password`。生产环境请改成真实地址和真实验收账号。

## 一键验收

测试命令：

```bash
API_BASE_URL=http://127.0.0.1:8000 API_USERNAME=owner API_PASSWORD=password bash scripts/check_api.sh
```

预期结果：

- 末尾显示 `验收通过：所有检查项均正常。`
- 任意一项显示 `FAIL` 都不能算通过。

失败时怎么排查：

- 先看失败项后面的 HTTP 状态码和返回内容。
- 如果连接失败，检查服务是否启动、端口是否正确、Nginx 或后端地址是否写错。
- 如果登录失败，检查账号密码、用户是否停用、数据库默认账号是否已初始化。
- 如果健康检查失败，继续按下面“服务器、数据库、Redis”三项排查。

## 1. 登录是否成功

测试命令：

```bash
curl -i -X POST http://127.0.0.1:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"owner","password":"password"}'
```

预期结果：

- HTTP 状态码为 `200`。
- 返回内容包含 `"ok":true` 和 `"token"`。

失败时怎么排查：

- `401`：账号或密码错误，或用户被停用。
- `500`：检查后端日志、数据库连接、Redis 连接。
- 没有返回：检查后端服务和 Nginx 是否启动。

## 2. 服务器是否正常

测试命令：

```bash
curl -s http://127.0.0.1:8000/api/health
```

预期结果：

- 返回内容包含 `"status":"running"`。

失败时怎么排查：

- 检查后端进程或 Docker 容器是否运行。
- 检查端口是否正确。
- 检查 Nginx 转发配置是否指向后端服务。

## 3. 数据库是否正常

测试命令：

```bash
curl -s http://127.0.0.1:8000/api/health
```

预期结果：

- 返回内容包含 `"database":true`。

失败时怎么排查：

- 检查 PostgreSQL 容器或数据库服务是否运行。
- 检查 `.env` 中 `DATABASE_URL` 是否正确。
- 检查数据库用户名、密码、库名是否和部署配置一致。
- 检查 Alembic 迁移是否已执行：`alembic upgrade head`。

## 4. Redis 是否正常

测试命令：

```bash
curl -s http://127.0.0.1:8000/api/health
```

预期结果：

- 返回内容包含 `"redis":true`。

失败时怎么排查：

- 检查 Redis 容器或服务是否运行。
- 检查 `.env` 中 `REDIS_URL` 是否正确。
- 检查服务器防火墙或容器网络是否阻断 Redis 连接。

## 5. 老板驾驶舱是否能打开

测试命令：

```bash
curl -i http://127.0.0.1:8000/control.html
curl -i -H "Authorization: Bearer <登录返回的token>" http://127.0.0.1:8000/api/dashboard
```

预期结果：

- 页面请求返回 HTTP `200`。
- 接口请求返回 HTTP `200`，且返回老板驾驶舱相关 JSON。

失败时怎么排查：

- 页面 `404`：检查 `frontend/control.html` 是否存在，后端静态页面路由是否正常。
- 接口 `401`：重新登录并替换 token。
- 接口 `500`：检查后端日志、数据库连接和用户权限数据。

## 6. 京东60店数据中心是否能打开

测试命令：

```bash
curl -i http://127.0.0.1:8000/jd-dashboard.html
curl -i -H "Authorization: Bearer <登录返回的token>" http://127.0.0.1:8000/api/stores
```

预期结果：

- 页面请求返回 HTTP `200`。
- 店铺接口返回 HTTP `200`，返回内容是店铺列表 JSON。

失败时怎么排查：

- 页面打不开：检查前端文件和后端静态路由。
- 店铺接口 `401`：检查 token。
- 店铺接口 `403`：检查当前账号是否有店铺或京东数据权限。
- 返回空列表：检查是否已初始化京东店铺数据。

## 7. 今日数据录入是否能打开

测试命令：

```bash
curl -i http://127.0.0.1:8000/metrics.html
curl -i -H "Authorization: Bearer <登录返回的token>" http://127.0.0.1:8000/api/jd/metrics/summary
```

预期结果：

- 页面请求返回 HTTP `200`。
- 指标汇总接口返回 HTTP `200`，返回今日 GMV、利润、广告费等字段。

失败时怎么排查：

- 页面 `404`：检查 `frontend/metrics.html` 是否存在。
- 接口 `401`：重新登录。
- 接口 `500`：检查 `jd_daily_metrics` 表和数据库迁移状态。

## 8. Excel导入是否能打开

测试命令：

```bash
curl -i http://127.0.0.1:8000/import.html
```

预期结果：

- 页面请求返回 HTTP `200`。

失败时怎么排查：

- 检查 `frontend/import.html` 是否存在。
- 检查 Nginx 或后端页面路由是否正常。
- 如果页面打开但导入失败，再检查 `/api/metrics/import` 接口和上传文件格式。

## 9. 京东商智采集接口是否存在

测试命令：

```bash
curl -s http://127.0.0.1:8000/openapi.json | grep '/api/jd/sync/store/{store_id}'
```

预期结果：

- 能看到 `/api/jd/sync/store/{store_id}`。
- 该接口是 `POST`，用于提交单店京东采集任务。

失败时怎么排查：

- 检查后端是否加载 `jd_collection` 路由。
- 检查代码中是否存在 `POST /api/jd/sync/store/{store_id}`。
- 检查后端启动日志是否有导入错误。

## 10. 京准通广告采集接口是否存在

测试命令：

```bash
curl -s http://127.0.0.1:8000/openapi.json | grep '/api/jd/sync/store/{store_id}'
curl -s http://127.0.0.1:8000/openapi.json | grep '/api/jd/sync/all'
```

预期结果：

- 能看到 `/api/jd/sync/store/{store_id}` 和 `/api/jd/sync/all`。
- 单店同步接口会创建京东商智和京准通两个采集任务；全店同步接口会批量创建采集任务。

失败时怎么排查：

- 检查采集路由是否注册。
- 检查 Redis 队列配置是否正常。
- 检查 worker 是否启动。注意：本项只验证接口存在，不要求真的触发采集。

## 11. 服务器重启后是否自动恢复

测试命令：

```bash
docker compose ps
docker compose restart backend worker nginx
sleep 10
bash scripts/check_api.sh
docker compose ps
```

预期结果：

- `backend`、`worker`、`nginx` 重启后仍为运行状态。
- `scripts/check_api.sh` 仍然全部通过。
- `docker compose ps` 中关键服务没有反复退出。

失败时怎么排查：

- 检查 `docker-compose.yml` 中服务是否配置 `restart: unless-stopped`。
- 检查后端启动日志，重点看数据库迁移、数据库连接、Redis 连接。
- 检查 worker 日志，确认队列消费进程能启动。
- 如果重启后短暂失败，等待数据库和 Redis 健康检查完成后再重试。
