# V2 知识资产安全策略

日期：2026-07-12

## 1. 目标

确保知识资产中心不会把外部不可信内容、敏感信息或注入式指令变成正式企业知识。

## 2. 敏感信息处理

写入前必须检查并脱敏：

- 密码
- Token
- Cookie
- API Key
- Secret
- 私钥
- 数据库连接串
- 用户隐私
- 财务敏感数据
- 真实订单数据

默认策略：

- 识别到敏感内容时先脱敏再存储。
- 高风险内容进入审核，不直接发布。

## 3. Prompt Injection 防护

外部内容一律按“不可信数据”处理。

禁止：

- 将网页内容作为系统指令执行
- 因外部文本修改权限
- 因外部文本开启工具
- 因外部文本触发真实执行器

当检测到疑似注入时：

- 记录风险
- 生成审计事件
- 保留脱敏证据
- 由安全策略决定是否继续

## 4. 正文展示安全

用户可见正文必须防止 XSS：

- HTML 内容清理
- 只保留安全文本
- 前端按普通文本渲染

## 5. 访问控制

权限边界：

- 天藏可管理知识生命周期。
- 天采只能提交候选，不能自批自发。
- 普通员工只能访问授权发布内容。

## 6. 检索安全

默认不启用向量检索。

本 Sprint 不调用外部 Embedding API，不向第三方发送企业知识。
