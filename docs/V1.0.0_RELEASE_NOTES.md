# 天统AI云中台 V1.0.0 Release Notes

## 1. 版本定位

版本：

```text
V1.0.0
```

定位：

```text
天统AI云中台第一个内部稳定基线版本。
```

V1.0.0 面向内部试运行，核心目标是让老板可以登录系统、查看企业经营状态、查看 AI员工、进入员工详情、管理任务流，并保持安全边界清晰。

## 3. 系统架构

核心组件：

- `frontend/`：静态 HTML 前端页面。
- `backend/`：FastAPI 后端。
- `backend/routers/`：模块化 API Router。
- `backend/services/`：只读聚合和业务服务。
- `backend/models.py` 与扩展模型文件：SQLAlchemy ORM。
- `alembic/`：数据库迁移。
- PostgreSQL：业务数据存储。
- Redis：队列、缓存和运行状态。
- Worker：后台任务处理。
- Nginx：前端静态服务和 API 反向代理。
- Docker Compose：本地和生产部署基础。

## 5. 使用流程

老板使用主流程：

```text
登录
-> 老板驾驶舱
-> AI员工中心
-> 点击 AI员工
-> 查看 AI员工详情
-> 进入 Task Center 查看或处理任务
```

新人本地启动：

```bash
cp .env.example .env
docker compose up -d --build
docker compose ps
```

访问：

```text
http://127.0.0.1/
```

健康检查：

```bash
curl http://127.0.0.1/api/health
curl http://127.0.0.1/api/ready
```

## 6. 安全边界

V1.0.0 保持：

```text
readonly=true
boss_confirm=true
security_audited=true
```

安全原则：

- AI员工中心只读展示。
- 高风险动作必须老板确认。
- 高风险动作必须安全审计。
- 不允许 AI员工自动提升权限。
- 不允许 AI员工自动升级技能。
- 不允许 AI员工绕过 Task Center 自动执行业务动作。
- 不接入 OpenClaw。
- 不接入 n8n。
- 不新增自动执行入口。

## 7. 已知限制

- V1.0.0 是内部试运行版本，不是完全自动驾驶系统。
- AI员工中心主要以只读展示和人工确认为主。
- 真实业务数据接入仍需逐步完成。
- 京东真实接口、OpenClaw、n8n 未接入。
- Execution Engine 不作为 AI员工中心自动执行入口。
- Docker 开发环境需要先复制 `.env.example` 为 `.env`。
- 发布安全测试要求根目录不保留真实 `.env`。
- 当前部分历史 Sprint 文件仍处于未提交状态，正式打 tag 前需要完成提交范围确认。
