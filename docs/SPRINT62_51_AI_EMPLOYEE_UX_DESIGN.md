# Sprint62.51 AI员工成长系统 UX 简化设计

## 1. 设计目标

把 `AI员工成长系统` 从“功能页面”进一步简化为老板每天能看懂的经营型页面。

老板打开页面后，3秒内回答一个问题：

> 我的 AI 公司现在怎么样？

第一屏只表达三件事：

- AI员工是否正常运行
- 今天做了多少任务
- AI员工整体有没有成长

本阶段只做 UX 设计，不写代码，不修改前端，不修改后端，不修改数据库。

## 2. 设计原则

### 2.1 3秒看懂

第一屏必须直接给结论，不让老板分析字段。

示例：

```text
🟢 27个AI员工运行正常
📈 今天完成128个任务
⭐ 今日成长评分92分
```

如果有风险：

```text
🟡 3个AI员工需要关注
📈 今天完成96个任务
⭐ 今日成长评分78分
```

如果系统异常：

```text
🔴 AI员工成长数据暂不可用
请稍后查看，当前不会自动执行任何动作
```

### 2.2 不使用后台管理语言

页面不展示：

- API
- ID
- 数据库字段
- 技术状态
- 内部枚举
- 原始 JSON
- 密集表格

页面使用老板能理解的话：

- “正常运行”
- “等待确认”
- “需要关注”
- “做了什么”
- “学会什么”
- “成长哪里”

### 2.3 员工展示用卡片

员工不使用表格。

员工卡片示例：

```text
┌──────────────┐
│ 天采          │
│ 数据部门      │
│ 🟢 运行中     │
│ 成长 92 分    │
└──────────────┘
```

卡片只展示：

- 员工名称
- 部门
- 状态
- 成长评分
- 风险提示

不展示：

- employee_id
- API字段
- 数据库字段
- 原始状态码

## 3. 页面结构设计

目标页面：

```text
frontend/ai-employee-growth-system.html
```

页面结构：

```text
顶部状态栏
  ├── 天统AI员工成长
  ├── 当前组织
  └── 只读安全模式

第一屏：AI公司今日状态
  ├── 运行结论
  ├── 今日完成任务
  ├── 今日成长评分
  └── 待Boss确认

员工成长卡片
  ├── 员工头像
  ├── 员工名称
  ├── 部门
  ├── 当前状态
  ├── 成长评分
  └── 风险提示

员工成长档案
  ├── 它负责什么
  ├── 最近做什么
  ├── 能力增加什么
  └── 需要关注什么

成长时间线
  ├── 最近完成任务
  ├── 最近沉淀经验
  ├── 最近风险提醒
  └── Boss确认记录

安全提示
  ├── 只读展示
  ├── 不自动执行
  ├── 不自动升级技能
  └── 不自动修改权限
```

## 4. 第一屏设计

### 4.1 视觉目标

第一屏像“AI公司体检结果”，而不是后台列表。

推荐结构：

```text
┌────────────────────────────────────────────┐
│ 🟢 AI公司运行正常                           │
│ 27个AI员工在线，今天完成128个任务，成长评分92分 │
└────────────────────────────────────────────┘

┌────────────┐ ┌────────────┐ ┌────────────┐
│ AI员工       │ │ 完成任务     │ │ 成长评分     │
│ 27名        │ │ 128个       │ │ 92分        │
└────────────┘ └────────────┘ └────────────┘
```

### 4.2 状态文案规则

| 状态 | 文案 |
| --- | --- |
| 正常 | `🟢 AI员工运行正常` |
| 有待确认 | `🟡 有事项等待Boss确认` |
| 有风险 | `🔴 有AI员工需要关注` |
| 无数据 | `当前暂无成长数据` |
| API异常 | `当前成长数据暂不可用` |

### 4.3 数据映射

| 页面展示 | 数据来源 |
| --- | --- |
| AI员工数量 | `/api/ai-employee-growth/overview.employees.total` |
| 完成任务 | `/api/ai-employee-growth/overview.tasks.completed` |
| 待确认 | `/api/ai-employee-growth/overview.tasks.waiting_confirm` |
| 成长评分 | `/api/ai-employee-growth/overview.average_growth.score` |
| 风险状态 | `/api/ai-employee-growth/overview.risk.high / medium` |

页面只转换成自然语言，不展示原始字段名。

## 5. 员工卡片设计

### 5.1 卡片字段

展示：

- 头像：使用员工名称首字或简洁头像
- 名称：如 `天采`
- 部门：如 `数据部门`
- 状态：如 `🟢 运行中`
- 成长评分：如 `成长92分`
- 风险提示：如 `正常`、`需要关注`

不展示：

- employee_id
- employee_code
- 数据库字段
- JSON字段
- 技术枚举

### 5.2 卡片布局

```text
┌─────────────────┐
│  天              │
│  天采            │
│  数据部门         │
│  🟢 运行中        │
│  ⭐ 成长92分      │
│  查看档案         │
└─────────────────┘
```

### 5.3 卡片状态

| 数据状态 | 展示文案 |
| --- | --- |
| active / working | `🟢 运行中` |
| idle | `⚪ 空闲` |
| waiting_confirm | `🟡 等待确认` |
| frozen / high_risk | `🔴 需要关注` |
| unknown | `暂无状态` |

## 6. 点击员工后的 AI员工档案

点击员工卡片进入员工档案区。

档案区不做后台表单，不做复杂详情页。

展示三个问题：

### 6.1 它负责什么

示例：

```text
天采负责数据采集、数据清洗和市场数据观察。
```

数据来源：

- 员工部门
- 员工岗位
- 员工职责
- 技能摘要

### 6.2 最近做什么

示例：

```text
最近完成了12个任务，其中10个通过Boss确认。
还有2个事项等待确认。
```

数据来源：

- `/api/ai-employee-growth/employees/{employee_id}.task_summary`
- `/api/ai-employee-growth/employees/{employee_id}/timeline`

### 6.3 能力增加什么

示例：

```text
沉淀了3条成功经验，记录了1条失败复盘。
成长评分从稳定区间继续提升。
```

数据来源：

- memory summary
- growth summary
- timeline growth event

### 6.4 需要关注什么

示例：

```text
当前无高风险。
```

或：

```text
有1条失败复盘，建议Boss查看原因。
```

注意：

- 只展示提醒。
- 不提供处理按钮。
- 不提供自动修复。
- 不提供自动执行。

## 7. 删除复杂信息

页面隐藏以下信息：

- API路径
- employee_id
- task_id
- 数据库表名
- JSON字段名
- 原始状态码
- 权限内部字段
- debug 文案
- 技术安全字段明细

安全信息保留为自然语言：

```text
当前页面只读展示，不会自动执行任务，不会自动升级技能，不会修改权限。
```

## 8. 空数据设计

### 8.1 全局无数据

```text
当前暂无成长数据
AI员工完成任务后，这里会显示成长情况。
```

### 8.2 员工无数据

```text
这个AI员工还没有成长记录
完成任务并通过Boss确认后，会开始形成成长档案。
```

### 8.3 API异常

```text
当前成长数据暂不可用
页面不会自动执行任何动作，请稍后再看。
```

## 9. 权限与安全边界

保持不变：

- 不修改登录系统
- 不修改 Boss Dashboard
- 不修改 Task Center
- 不修改 Orchestrator
- 不新增 POST 接口
- 不接 Execution Engine
- 不接 OpenClaw
- 不接 n8n
- 不自动执行任务
- 不自动升级技能
- 不自动修改权限

页面只允许：

- 查看总览
- 查看员工卡片
- 查看员工档案
- 查看成长记录

高风险规则：

- `boss_confirm=true`
- `security_audited=true`
- 页面只展示“需要Boss确认”，不提供自动处理入口。

- 页面不出现密集表格。
- 员工使用卡片展示。
- 点击员工能看到：
  - 它负责什么
  - 最近做什么
  - 能力增加什么
- 页面无执行按钮。
- 页面无技能升级按钮。
- 页面无权限修改入口。
- 不影响登录、Boss Dashboard、Task Center、Orchestrator。
