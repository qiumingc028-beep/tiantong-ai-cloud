# Sprint62.62 AI员工中心 V1 极简化体验优化方案

## 1. 阶段边界

本阶段只输出设计和小范围优化方案，不修改代码。

禁止：

- 修改代码
- 修改数据库
- 新增 API
- 修改 Task Center
- 接入自动执行
- 接入 OpenClaw / n8n

## 2. 已读取文件

- `README.md`
- `docs/SPRINT62_61_AI_WORKFORCE_PRODUCT_REVIEW.md`
- `frontend/ai-workforce.html`
- `frontend/ai-employee-detail.html`

## 3. 产品原则

目标标准：

```text
3岁小孩也能看懂。
```

具体原则：

1. 一个页面只回答一个问题。
2. 一个页面只保留一个主要动作。
3. 少数字，少标签，少说明。
4. 不出现技术词。
5. 不出现后台感。
6. 不出现执行、升级、授权等危险动作。

## 4. 首页极简化方案

### 4.1 首页只回答一个问题

当前问题：

```text
我的AI公司现在怎么样？
```

保留。

### 4.2 首页第一屏保留内容

只保留三类信息：

1. AI员工数量
2. 当前运行状态
3. 找员工入口

建议第一屏结构：

```text
我的AI公司现在怎么样？

绿色状态卡：
AI员工正常工作

27 个 AI员工
12 个正在工作

下面是你的AI员工
```

### 4.3 首页删除或下沉内容

删除或下沉：

- `当前组织：天统AI`
- `Sprint62.60`
- `只读模式`
- `老板确认`
- `安全审计`
- `已经完成`
- 复杂状态说明

其中安全信息不删除，只放到底部：

```text
只看不操作。所有重要动作都需要老板确认。
```

### 4.4 首页导航简化

当前侧边导航：

- 企业大脑总控台
- AI员工中心
- AI员工成长
- AI员工名册

建议 V1 首页只突出：

- 企业大脑
- AI员工中心

其他入口不删除，只弱化或后续放到更多入口。

## 5. 员工卡片极简化方案

### 5.1 卡片只显示四项

保留：

- 头像
- 名字
- 做什么
- 当前状态

建议卡片格式：

```text
圆形头像
天采
做数据采集
绿色：正在工作
```

### 5.2 卡片唯一动作

整张卡片可以点击进入详情。

不要再额外增加按钮。

保留文案：

```text
看详情
```

或直接整卡可点。

### 5.3 卡片禁止出现

- 技术字段
- API 信息
- 数据字段
- 工程状态
- 风险代码
- 任务状态码
- 权限入口
- 执行入口

## 6. 员工详情极简化方案

### 6.1 详情页只回答一个问题

详情页第一屏只回答：

```text
这个AI员工今天怎么样？
```

### 6.2 详情页第一句话

建议顶部用一句话替代多个指标：

```text
天采负责数据采集，今天完成 28 件事，当前正在工作。
```

这句话应放在员工头像和姓名下面。

### 6.3 四个模块命名

当前命名中，`他是谁` 还可以继续老板化。

建议改为：

- 我负责什么
- 我今天做了什么
- 我学会了什么
- 我的成长

不要使用：

- Skill
- API
- Memory
- Timeline
- Growth
- Audit
- Service
- Router

### 6.4 详情页第一屏内容

第一屏只保留：

```text
头像
员工名称
一句话说明
我负责什么
我今天做了什么
我学会了什么
我的成长
返回AI员工中心
```

### 6.5 详情页下方内容处理

当前详情页下方有：

- 我的能力
- 成长记录
- 成长时间线
- 成长评分
- 经验记录

建议：

1. V1 默认折叠。
2. 或移到第二屏。
3. 不在首屏抢占注意力。
4. 文案继续口语化：

| 当前文案 | 建议文案 |
| --- | --- |
| 我的能力 | 我擅长什么 |
| 成长时间线 | 最近变化 |
| 成长评分 | 成长分 |
| 技能等级 | 熟练程度 |
| 使用次数 | 用过几次 |
| 成功率 | 做成比例 |

## 7. 空状态与异常状态文案

### 7.1 首页空状态

当前：

```text
当前暂无AI员工数据
```

建议：

```text
现在还没有AI员工。
```

### 7.2 详情空状态

当前：

```text
暂无数据
```

建议按模块分别显示：

- 还没有安排工作
- 还没有学习记录
- 还没有成长记录
- 还没有需要老板看的问题

### 7.3 异常状态

当前：

```text
当前数据暂不可用
```

建议：

```text
现在看不到数据，请稍后再看。
```

## 8. 颜色规则

保持三色：

- 绿色：正常
- 黄色：需要看看
- 红色：有问题

文案建议：

| 状态 | 文案 |
| --- | --- |
| working | 正在工作 |
| idle | 正常待命 |
| frozen | 需要老板看看 |
| offline | 暂时离线 |

## 9. 安全表达方案

安全规则必须保留，但不放在老板第一眼核心区域。

当前：

- 只读模式
- 老板确认
- 安全审计

建议展示：

```text
只看不操作
```

底部说明：

```text
这个页面只查看AI员工状态。所有重要动作都需要老板确认，并保留安全记录。
```

内部仍保留：

- `readonly=true`
- `boss_confirm=true`
- `security_audited=true`

## 10. 小范围优化拆分

### Sprint62.63-A：首页极简化

修改范围：

- `frontend/ai-workforce.html`
- 对应前端测试

目标：

- 第一屏只保留 AI员工数量、运行状态、找员工入口。
- 顶部安全标签改为 `只看不操作`。
- 员工卡片继续只显示头像、名字、做什么、状态。

### Sprint62.63-B：详情页语言极简化

修改范围：

- `frontend/ai-employee-detail.html`
- 对应前端测试

目标：

- 第一屏改为一句话状态。
- 模块名改为：
  - 我负责什么
  - 我今天做了什么
  - 我学会了什么
  - 我的成长
- 下方复杂信息默认放到第二屏。

### Sprint62.63-C：极简体验验收

检查：

- 是否仍有技术词
- 是否仍有复杂菜单
- 是否仍有表格
- 是否仍有多个按钮
- 是否仍无执行入口

## 11. 验收标准

通过标准：

1. 首页 3 秒内看懂当前 AI公司状态。
2. 老板知道去哪里找 AI员工。
3. 员工卡片不需要解释。
4. 员工详情第一句话能说明这个员工今天怎么样。
5. 页面不出现 Skill、API、Memory、Timeline 等技术词。
6. 页面不出现执行、授权、升级、自动运行入口。
7. 安全边界仍保留：
   - `readonly=true`
   - `boss_confirm=true`
   - `security_audited=true`

## 12. 结论

Sprint62.62 建议继续做小范围前端体验优化，不扩大功能。

下一步应优先处理：

1. 首页第一屏再减信息。
2. 详情页第一屏一句话说明。
3. 技术词替换为普通人语言。
4. 安全信息从顶部技术标签改为底部说明。

不建议进入：

- 新 API
- 数据库改动
- Task Center 改动
- 自动执行
- OpenClaw / n8n 接入
