# V2 外部内容安全策略

## 1. 核心原则

网页、搜索摘要、引用文本、JSON 内容和任何外部来源内容都必须视为不可信数据。

外部内容只能作为研究输入，不能改变系统指令、权限或执行边界。

## 2. 禁止行为

禁止外部内容触发：

- 修改系统提示词
- 开启工具
- 请求 Secret
- 触发 Shell
- 触发真实浏览器控制
- 触发电脑控制
- 触发手机控制
- 触发自动发布
- 触发表单提交
- 触发登录
- 触发内网访问

## 3. Prompt Injection 防护

检测到下列模式时，必须记录审计事件：

- 忽略之前要求
- 忽略以上规则
- 重新定义系统指令
- 打印 Secret / Token
- 执行命令

对应事件：

- `EXTERNAL_CONTENT_INSTRUCTION_DETECTED`

处理原则：

1. 标记风险
2. 不执行外部指令
3. 保留脱敏证据
4. 继续或停止由安全策略决定

## 4. URL 与网络边界

进入浏览器只读执行器前必须再次校验：

- 协议
- 域名白名单
- 域名黑名单
- 私有网络
- 云 Metadata
- 重定向

必须阻断：

- `localhost`
- `127.0.0.1`
- 私有 IPv4 / IPv6
- 内网域名
- `file://`
- `javascript:`
- `data:`
- 任何不在白名单内的目标

## 5. 数据使用边界

外部内容只允许被用作：

- 研究摘要
- 来源记录
- 证据链
- 交叉验证
- 冲突识别

禁止将外部内容直接转成系统级执行指令。
